# 金字塔概念图示例

## 场景说明

金字塔概念图适用于展示层次结构、优先级、从基础到高级的递进关系。常见应用场景包括：

- **需求层次**：马斯洛需求层次理论
- **产品层级**：基础功能 → 高级功能 → 旗舰功能
- **市场细分**：大众市场 → 中端市场 → 高端市场
- **技术成熟度**：基础技术 → 应用技术 → 前沿技术
- **价值主张**：基础价值 → 核心价值 → 增值服务

## 设计规范

### McKinsey 风格配色

- **顶层（第 1 层）**：主强调色 `#F85d42`（橙红色）
- **中层（第 2 层）**：深蓝色 `#556EE6`
- **底层（第 3 层）**：绿色 `#34c38f`

### 三角形层级设计

使用 CSS `clip-path` 属性创建真正的三角形形状：

```css
/* 顶层 - 倒三角形 */
clip-path: polygon(50% 0%, 0% 100%, 100% 100%);

/* 中层 - 梯形 */
clip-path: polygon(25% 0%, 75% 0%, 100% 100%, 0% 100%);

/* 底层 - 宽梯形 */
clip-path: polygon(16.67% 0%, 83.33% 0%, 100% 100%, 0% 100%);
```

### 尺寸规范

| 层级 | 宽度 | 高度 | 字体大小 |
|------|------|------|----------|
| 顶层 | 140px | 80px | 13px |
| 中层 | 280px | 90px | 15px |
| 底层 | 420px | 100px | 17px |

### 交互效果

- **悬停效果**：放大 1.05 倍 + 增强阴影
- **过渡动画**：0.3s 缓动
- **光标样式**：pointer

## 使用建议

1. **层次数量**：建议 3-5 层，过多会导致视觉拥挤
2. **文本长度**：每层文字控制在 10-15 字以内
3. **色彩选择**：从上到下使用渐变色或对比色
4. **响应式**：在小屏幕上自动缩小尺寸，保持可读性
5. **数据标注**：可在每层添加关键数据指标（如增长率、占比等）

## 适用场景

- 市场细分与定位
- 产品功能层级展示
- 技术发展路线图
- 组织架构层次
- 需求优先级排序
- 价值主张分层

## 不适用场景

- ❌ 平等关系的数据（应使用饼图、环形图）
- ❌ 时间序列数据（应使用折线图、柱状图）
- ❌ 多维对比（应使用雷达图）
- ❌ 流程流向（应使用流程图、桑基图）
