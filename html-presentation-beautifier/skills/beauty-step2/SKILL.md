# Beauty 步骤2：幻灯片内容转换与拆分

## 目标

将文档内容完整转换为结构化的幻灯片页面，确保内容零遗漏、逻辑清晰、页面适量。

## 核心原则

### 绝对禁止规则 [CRITICAL]

**⚠️ 重要警告：禁止节省token、禁止缩减内容、禁止偷工减料**

- ✅ **100%保留原文所有内容，零遗漏、零压缩、零简化**
- ✅ **遇到内容过多时，必须使用"继续"机制分批发送完整内容**
- ✅ **每个要点必须完整保留，禁止用省略号、省略、简化表述替代**
- ✅ **所有数据点（数字、百分比、金额等）必须完整保留**
- ✅ **所有专业术语、完整描述必须完整保留**

### NEVER规则

1. **NEVER节省token**
   - 如果内容太长，一次无法完整输出，必须使用"继续"机制分批输出
   - 禁止为了减少输出量而压缩句子、合并要点、删减描述
   - 宁可分10页输出完整内容，也不要在1页中简化内容

2. **NEVER缩减内容**
   - 原文有多少字，输出就必须保留多少字（除非是纯格式符号）
   - 禁止用"等"、"略过"、"详见前文"等表述替代具体内容
   - 禁止将多个要点合并成一个复合句

3. **NEVER偷工减料**
   - 每个数据点都必须完整保留（如：$1,234,567不能简化为"超过100万"）
   - 每个专业术语都必须保留完整（如：不能将"亚马逊A9算法"简化为"算法"）
   - 每个例子、案例都必须完整保留

### 分页处理规则

当单个步骤的输出内容超过单次输出限制时：

```
阶段a：输出前半部分内容
→ 输出："[步骤X阶段a完成] - 已输出前N个要点
       剩余M个要点待输出
       请输入'继续'以获取剩余内容"

【等待用户输入"继续"】

阶段b：输出后半部分内容
→ 输出："[步骤X阶段b完成] - 已输出全部N+M个要点
       内容完整性：100%
       已进入下一步"
```

### 页面拆分规则

- ✅ 每页内容不超过8个要点，超过则必须分页
- ✅ 分页时每个页面的内容必须完整保留
- ✅ 宁可增加页数，绝不简化内容

## 执行流程

```
步骤2.1：识别文档结构层次 → 步骤2.2：规划幻灯片页面类型 → 步骤2.3：拆分内容到具体页面 → 步骤2.4：验证内容完整性
```

## 步骤 2.1：识别文档结构层次

### 目标

### 执行要求

#### 阶段1：提取标题层级

基于步骤1的分析结果，提取完整的标题结构：

```
文档结构地图：

H1: 主标题（文档总标题）
  ├─ H2: 第一章节
  │   ├─ H3: 子章节1.1
  │   │   ├─ 要点1
  │   │   ├─ 要点2
  │   │   └─ 要点3
  │   ├─ H3: 子章节1.2
  │   └─ H3: 子章节1.3
  ├─ H2: 第二章节
  └─ H2: 第三章节
```

#### 阶段2：统计内容密度

为每个章节统计内容量：

```
内容密度统计：

第一章节（H2）：要点 6个，数据点 7个，表格 1个
第二章节（H2）：要点 20个，数据点 3个，表格 2个
第三章节（H2）：要点 4个，数据点 1个，表格 0个
```

#### 阶段3：识别特殊内容块

标记需要特殊处理的内容：

- □ 对比分析（A vs B）
- □ 时间序列（2024-2026）
- □ 数据表格（≥3列）
- □ 流程步骤（1→2→3）
- □ 案例故事
- □ 强调结论："关键发现"、"核心建议"

### Token限制处理

如果文档包含10+个H2章节，必须分批识别：

```

→ 输出："步骤2.1阶段1a完成 - 已识别前5个章节结构

【等待用户输入"继续"】

→ 输出："步骤2.1阶段1b完成 - 章节结构100%识别完成
       已进入步骤2.2"
```

### 输出产物

- 完整的文档结构地图（所有H1/H2/H3）
- 内容密度统计表
- 特殊内容标记清单

- [ ] 内容密度已统计

- [ ] 无遗漏章节

## 步骤 2.2：规划幻灯片页面类型

### 目标

根据文档结构，规划所有幻灯片页面的类型和数量。

### 幻灯片页面类型

#### P1. 封面页
- 数量：1页
- 内容：主标题、副标题（如有）、日期或作者（如有）
- 模板：`01-cover-page.html`

#### P2. 目录页
- 数量：1-2页
- 分页规则：≤6章节→1页，7-12章节→2页，≥13章节→分组目录
- 模板：`08-table-of-contents.html`

#### P3. 章节首页
- 数量：每个H2章节1页
- 内容：章节标题、章节编号、章节描述、子章节导航（可选）
- 模板：`11-chapter-overview.html`

#### P4. 内容页 [核心]
- 数量：根据内容密度动态计算
- 标题：页面标题（显示在导航栏左侧）
- 要点：≤8个要点/页

#### P5. 结束页
- 数量：1页
- 内容："谢谢"、联系方式、下一步行动（如有）

### 布局选择决策树

```
                    ┌─────────────────────┐
                    │  分析内容类型和数量   │
                    └──────────┬──────────┘
                               │
                ┌──────────────┼──────────────┐
                │              │              │
           纯文本型         图文混排型      数据展示型
                │              │              │
        ┌───────┴───────┐      │      ┌───────┴───────┐
        │               │      │      │               │
    ≤8要点        >8要点    │   单图表      多图表
        │               │      │      │               │
    单列布局    分页处理    │  两列布局   三列布局
                                   （图表+洞察）（并排对比）
```

### 关键规则 [CRITICAL]

1. **分页规则**：每页最多8个要点
   - 9-16个要点 → 2页（标题加序号：一、二）
   - 17-24个要点 → 3页（标题加序号：一、二、三）

2. **布局优先原则**：
   - 第一步：分析内容特征（要点数、图表数、表格数）
   - 第二步：选择布局类型
   - 第三步：分配内容到页面

3. **图表布局规则**：
   - ❌ 禁止单列布局放置图表
   - ✅ 必须使用2列布局（图表+洞察/摘要）
   - ✅ 或使用3列布局（多个图表并排对比）

### 执行流程

#### 阶段1：规划固定页面

```
固定页面规划：
1. 封面页：1页
2. 目录页：X页（根据H2章节数）
3. 结束页：1页
→ 固定页面总计：[1 + X + 1] 页
```

#### 阶段2：规划章节首页

```
章节首页规划：每个H2章节1页
→ 章节首页总计：[N个H2] 页
```

#### 阶段3：规划内容页

**核心计算公式**：

```
单列布局页面容量 = 8个要点
两列布局页面容量 = 4个要点/列 × 2列 = 8个要点
三列布局页面容量 = 3个要点/列 × 3列 = 9个要点

每章内容页数 = ceil(该章要点总数 / 8)
```

### 输出产物

- 完整页面规划清单（类型、数量、标题）
- 每页布局类型分配
- 页面总数统计

## 步骤 2.3：拆分内容到具体页面

### 目标

将文档内容按照步骤2.2的规划，拆分到具体的幻灯片页面，并生成完整的内容描述和图表解释。

### 核心原则

**原则一：内容完整性优先**
- 100%保留原文所有要点，一个都不能少
- 禁止为了减少页数而压缩、合并或省略要点
- 如果要点超过8个，必须分页展示

**原则二：布局服务于内容**
- 先分析内容特征，再选择布局类型
- 布局选择必须能够容纳所有内容
- 禁止为了适应布局而削减内容

**原则三：逻辑分组优先**
- 保持相关内容的连贯性和逻辑性
- 按主题/分组拆分，而非机械切分

**原则四：内容丰富化原则 [NEW]**
- 每个内容页必须有完整的描述总结，不能只有要点列表
- 图表页必须有对应的解释说明，不能只有图表本身
- 信息表达必须完整，不能遗漏任何关键细节
- 内容页的价值在于传达完整的思想，而非简单的摘要罗列

**原则五：一致性原则**
- 同类型内容使用相同布局
- 相邻页面保持视觉一致性

### 内容分析方法

#### 步骤1：统计内容元素

```
子章节内容元素统计：

┌─────────────────────────────────────┐
│ 元素类型              │ 数量  │ 备注 │
├───────────────────────┼───────┼──────┤
│ 纯文本要点            │ ___个 │      │
│ 数据点（数字/百分比） │ ___个 │      │
│ 图表/可视化          │ ___个 │      │
│ 表格                 │ ___个 │      │
│ 对比项（A vs B）     │ ___组 │      │
│ 流程步骤（1→2→3）    │ ___步 │      │
└─────────────────────────────────────┘
```

#### 步骤2：分析内容复杂度

```
复杂度评估：

低复杂度：≤4个要点，无图表/表格 → 单列布局
中等复杂度：5-8个要点，或包含1个图表/表格 → 单列布局
高复杂度：9-16个要点，或包含2个图表/表格 → 分页或两列布局
极高复杂度：>16个要点，或包含3+图表/表格 → 多页拆分
```

#### 步骤3：识别内容分组

如果子章节内容包含明确的分组结构：

```
示例1：按主题分组
原内容："市场分析"
  - 分组1：市场规模（3个要点）
  - 分组2：增长趋势（4个要点）
  - 分组3：竞争格局（3个要点）
→ 识别为3个独立分组
```

### 列表样式选择

根据内容类型自动选择列表样式：

```
内容类型              │ 推荐样式              │ 识别关键词
─────────────────────┼───────────────────────┼─────────────────────
普通并列要点          │ 标准要点列表          │ 以及、另外、此外
步骤/流程            │ 编号列表              │ 首先、其次、最后
重要项目突出          │ 卡片列表              │ 核心、关键、重要
优劣势对比            │ 对比列表              │ 对比、差异、优劣
时间线/里程碑         │ 时间线列表            │ 年份、历程、发展
带标签分类            │ 图标列表              │ 类型、分类、维度
```

### 步骤 2.3.1：内容页描述总结生成 [NEW]

### 目标

为每个内容页生成完整的描述总结，确保页面信息表达完整、丰富。

### 核心要求

**必须为每个内容页生成的要素**：

1. **页面导语**（必须）
   - 用1-2段话概括本页面要表达的核心观点
   - 说明这些内容对整体报告的意义
   - 建立读者的阅读预期

2. **要点详细展开**（必须）
   - 每个要点不能只有标题，必须有完整的解释说明
   - 包含：背景描述、具体数据、影响分析、结论推导
   - 禁止：只保留要点标题，删除解释说明
   - 禁止：只保留数字，删除单位（如 "$"、亿"、"%"）
   - 禁止：只保留结论，删除论据和过程

3. **跨要点关联说明**（推荐）
   - 揭示多个要点之间的逻辑关系
   - 说明这些内容如何相互支撑
   - 引导读者理解整体思想

### 禁止行为 [CRITICAL]

**❌ 绝对禁止**：
- ❌ 只列出要点标题，没有详细解释
- ❌ 用"详见前文"替代具体内容
- ❌ 用"等"字省略后续细节
- ❌ 将完整描述压缩成一句话
- ❌ 删除案例中的具体细节
- ❌ 简化专业术语的解释

**✅ 正确做法**：
- ✅ 标题 + 完整描述必须同时保留
- ✅ 每个要点都有独立、完整的展开说明
- ✅ 所有数据点（数字、百分比、金额）必须完整保留
- ✅ 所有专业术语必须有必要的解释说明
- ✅ 案例细节必须完整保留

```
**详细内容**：

**要点一：市场规模庞大且持续增长**
根据最新市场调研数据，2024年北美返校季市场规模达到365.9亿美元，预计2025年将增长至394亿美元。这一增长趋势反映了家长对教育投资的持续重视，以及学生返校购物需求的刚性特征。市场的整体规模为我们的业务拓展提供了充足的市场空间，而持续增长则意味着市场机会的长期可持续性。
```

### 步骤 2.3.2：图表解释说明生成 [NEW]

### 目标

为每个包含图表的内容页生成完整的图表解释说明，确保读者能够充分理解图表传达的信息。

### 核心要求

**图表页必须包含的说明要素**：

1. **图表概述**（必须）
   - 清晰说明图表展示的主题和数据范围
   - 说明选择该图表类型的原因
   - 概括图表传达的核心发现

2. **数据解读**（必须）
   - 按数据系列或数据点逐项解读
   - 包含所有关键数据点的具体数值
   - 说明数据背后的含义和趋势

3. **洞察分析**（必须）
   - 从数据中提炼关键洞察
   - 说明这些洞察对业务决策的意义
   - 揭示数据中隐藏的机会或风险

4. **行动建议**（推荐）
   - 基于图表洞察提出具体建议
   - 说明建议与数据之间的关联

## 自由度校准

### 零自由度场景 [全部强制执行]

在内容处理过程中，以下场景**没有任何自由度**，必须严格按照规则执行：

#### 零自由度 - 内容保留
- ✅ 原文有多少字，输出就必须保留多少字
- ✅ 每个数据点（数字、百分比、金额）必须完整保留
- ✅ 每个专业术语必须完整保留
- ✅ 每个案例细节必须完整保留
- ✅ 每个要点必须独立成行，不能合并

#### 零自由度 - 分页处理
- ✅ 如果一次无法完整输出，必须使用"继续"机制
- ✅ 分页时宁可增加页数，绝不简化内容
- ✅ 每页不超过8个要点，超过必须分页

#### 零自由度 - 布局选择
- ✅ 图表禁止使用单列布局，必须使用2列或3列布局
- ✅ 相关联的内容必须在同一页
- ✅ 表格、图表、对比分析必须单独处理

### 执行优先级

当出现冲突时，按以下优先级执行：

1. **内容完整性 > 布局美观**
   - 宁可使用更多页面展示完整内容，也不简化任何内容
   
2. **数据准确性 > 表达简洁**
   - 宁可保留原始数据和单位，也不简化表达

3. **完整输出 > 单次输出**
   - 宁可分批使用"继续"机制，也不一次性简化输出
