# 术语定义

本文档提供幻灯片演示生成过程中使用的所有专业术语定义，供Agent理解和执行任务时参考。

---

## 1 文档结构术语

### 1.1 H1（主标题）

**定义**：文档的最高级别标题，通常是文档的总标题或主题名称。

**示例**：
```markdown
# 跨境电商行业发展报告
```

**说明**：在幻灯片中，H1对应封面页的主标题。一个文档只能有一个H1标题。

**关联概念**：H2、H3、章节、子章节。

---

### 1.2 H2（二级标题）

**定义**：文档的第二级别标题，用于标识文档的主要章节。

**示例**：
```markdown
# 跨境电商行业发展报告

## 第一章 市场规模分析
```

**说明**：H2标题通常对应幻灯片的章节首页（P3）。一个H1下可以有多个H2。

**关联概念**：H1、H3、章节、章节首页。

---

### 1.3 H3（三级标题）

**定义**：文档的第三级别标题，用于标识章节内的子章节。

**示例**：
```markdown
## 第一章 市场规模分析

### 1.1 总体市场规模
```

**说明**：H3标题通常对应幻灯片的内容页（P4）的页面标题。

**关联概念**：H1、H2、要点、内容页。

---

### 1.4 章节

**定义**：以H2标题开头的一段文档内容，包含该章节下的所有H3子章节和段落内容。

**说明**：在幻灯片结构中，每个H2章节对应一个章节首页（P3）和若干内容页（P4）。

**示例结构**：
```markdown
## 第二章 平台分析（章节开始）
### 2.1 平台A分析
内容...
### 2.2 平台B分析
内容...
## 第三章 用户画像（下一个章节开始）
```

---

### 1.5 子章节

**定义**：以H3标题开头的内容单元，通常包含若干个要点或段落。

**说明**：子章节是内容拆分的基本单位，每个子章节对应一个或多个内容页。

**示例**：
```markdown
### 1.1 市场概况
要点1
要点2
要点3
```

---

### 1.6 要点

**定义**：文档中的独立观点、陈述或数据点，通常以段落或列表项的形式呈现。

**说明**：要点是幻灯片内容页的基本组成单元。每个内容页最多包含8个要点。

**类型分类**：
- 纯文本要点：以文字形式表达的观点或陈述。
- 数据要点：包含具体数字、百分比或统计数据的陈述。
- 图表引用要点：需要配合图表展示的数据描述。

---

### 1.7 数据点

**定义**：文档中出现的具体数值、百分比、统计数据或其他量化信息。

**示例**：
- 「2024年中国跨境电商市场规模达15万亿元」
- 「年复合增长率保持25%以上」
- 「前三平台占比超60%」

**说明**：数据点是幻灯片内容的重要组成部分，必须完整保留，不能省略或简化。

---

## 2 幻灯片页面类型术语

### 2.1 封面页（Cover Page）

**定义**：幻灯片的首页，用于展示文档的主标题、作者、日期等基本信息。

**模板文件**：`01-cover-page.html`

**必要元素**：主标题（H1）。

**可选元素**：副标题、日期、作者、版本号、机构名称。

**示例**：
```
┌─────────────────────┐
│                     │
│     跨境电商         │
│   行业发展报告       │
│                     │
│     2024年12月      │
│     张三            │
└─────────────────────┘
```

---

### 2.2 目录页（Table of Contents）

**定义**：展示文档所有章节结构的页面，用于导航和概览。

**模板文件**：`08-table-of-contents.html`、`10-toc-grid-cards.html`

**分页规则**：
- ≤6个章节 → 1页（列表式或网格式）
- 7-12个章节 → 2页（每页6个）
- ≥13个章节 → 分组目录（按主题分2-3页）

**列表式示例**：
```
目录
1. 第一章 市场规模分析
2. 第二章 平台分析
3. 第三章 用户画像
```

**网格式示例**：
```
┌─────────────────┬─────────────────┐
│ 第一章          │ 第四章          │
│ 市场规模分析    │ 发展趋势预测    │
├─────────────────┼─────────────────┤
│ 第二章          │ 第五章          │
│ 平台分析        │ 风险与挑战      │
├─────────────────┼─────────────────┤
│ 第三章          │                 │
│ 用户画像        │                 │
└─────────────────┴─────────────────┘
```

---

### 2.3 章节首页（Chapter Cover）

**定义**：每个章节的起始页面，用于展示章节标题和章节概述。

**模板文件**：`11-chapter-overview.html`

**必要元素**：章节编号、章节标题、章节描述。

**可选元素**：子章节导航、章节图标。

**示例**：
```
┌─────────────────────┐
│                     │
│     第一章          │
│                     │
│   市场规模分析      │
│                     │
│   本章将分析跨境    │
│   电商的整体市场    │
│   规模和增长趋势    │
│                     │
│   1.1 总体市场规模  │
│   1.2 增长趋势      │
└─────────────────────┘
```

---

### 2.4 内容页（Content Page）

**定义**：展示具体内容要点的页面，是幻灯片的核心页面类型。

**模板文件**：`02-two-column-comparison.html`、`03-three-column.html`、`04-card-grid.html`、`05-chart-text.html`

**必要元素**：页面标题、要点列表。

**可选元素**：图表、表格、数据摘要。

**核心规则**：每页最多8个要点。

**单列布局示例**：
```
┌─────────────────────┐
│                     │
│   市场分析          │
│                     │
│  • 市场规模15万亿元 │
│  • 增长率25%        │
│  • 前三占比60%      │
│  • 用户画像25-40岁  │
│                     │
└─────────────────────┘
```

---

### 2.5 结束页（Closing Page）

**定义**：幻灯片的最后一页，用于致谢和提供后续联系方式。

**模板文件**：`99-closing.html`

**必要元素**：「谢谢」或「Thank You」。

**可选元素**：联系方式、社交媒体链接、下一步行动、Q&A提示。

**示例**：
```
┌─────────────────────┐
│                     │
│                     │
│        谢谢         │
│                     │
│     contact@        │
│       example.com   │
│                     │
│     Q&A             │
│                     │
└─────────────────────┘
```

## 3 布局类型术语

### 3.1 单列布局（Single Column Layout）

**定义**：所有内容在一列中垂直排列的布局方式。

**适用场景**：≤8个要点的纯文本内容。

**特点**：
- 内容自上而下排列
- 适合要点数量较少的内容
- 阅读顺序清晰

**页面容量**：8个要点。

**CSS Grid定义**：
```css
.single-column {
    display: grid;
    grid-template-columns: 1fr;
    gap: 35px;
}
```

---

### 3.2 两列布局（Two Column Layout）

**定义**：内容分为左右两列排列的布局方式。

**适用场景**：
- 图表配合数据洞察
- 对比型内容
- 左侧数据、右侧分析

**页面容量**：8个要点（每列4个）。

**变体形式**：
- 左图表右洞察（50%图表+50%洞察）
- 左数据摘要右详细说明（50%+50%）

**CSS Grid定义**：
```css
.two-column {
    display: grid;
    grid-template-columns: 1fr 1fr;
    gap: 35px;
}
```

---

### 3.3 三列布局（Three Column Layout）

**定义**：内容分为三列并排排列的布局方式。

**适用场景**：
- 多个图表并排对比
- 三个并列的主题
- 分类展示三个对象

**页面容量**：9个要点（每列3个）。

**CSS Grid定义**：
```css
.three-column {
    display: grid;
    grid-template-columns: 1fr 1fr 1fr;
    gap: 35px;
}
```

---

### 3.4 卡片网格布局（Card Grid Layout）

**定义**：以卡片形式展示多个独立内容的网格布局。

**适用场景**：
- 产品或服务介绍
- 多个并列的功能特性
- 团队成员介绍

**特点**：
- 每个卡片独立展示
- 支持响应式调整
- 视觉层次清晰

**CSS Grid定义**：
```css
.card-grid {
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
    gap: 25px;
}
```

## 4 内容处理术语

### 4.1 内容密度

**定义**：单位内容区域内包含的信息量，通常以要点数量、数据点数量和表格数量来衡量。

**计算方式**：
```
内容密度 = 要点数 + 数据点数 + 表格数 × 3
```

**密度等级**：
- 低密度：内容密度 ≤ 4
- 中等密度：内容密度 5-8
- 高密度：内容密度 9-16
- 极高密度：内容密度 > 16

**处理策略**：
- 低密度：单页单列布局
- 中等密度：单页可能需要分列
- 高密度：分页或使用多列布局
- 极高密度：必须分多页处理

---

### 4.2 内容拆分

**定义**：将大量内容按规则分配到多个幻灯片页面的过程。

**拆分原则**：
1. 每页不超过8个要点
2. 相关内容尽量放在同一页
3. 保持逻辑连贯性
4. 保持主题聚焦

**拆分策略**：
- 水平拆分：按要点数量平均分配
- 垂直拆分：按主题或分组分配
- 混合拆分：结合水平和垂直策略

---

### 4.3 逻辑分组

**定义**：识别内容中具有内在关联性的部分，将其视为一个整体进行处理。

**分组类型**：
- 按主题分组：围绕同一主题的内容归为一组
- 按时间分组：按时间顺序排列的内容归为一组
- 按对象分组：针对同一对象的不同方面归为一组
- 按对比分组：需要对比分析的内容归为一组

**示例**：
```markdown
原内容："市场分析"
  - 分组1：市场规模（主题分组）
  - 分组2：增长趋势（主题分组）
  - 分组3：竞争格局（主题分组）
```

---

### 4.4 继续机制

**定义**：当内容过多无法在单次交互中完成时，采用分批处理并等待用户确认的机制。

**触发条件**：
- 文档包含10个以上的H2章节
- 单次处理可能超出Token限制
- 用户明确要求分批处理

**执行流程**：
1. 处理前N个章节（如前5个）
2. 输出完成状态和提示信息
3. 等待用户输入「继续」
4. 处理剩余章节
5. 完成全部处理

## 5 验证术语

### 5.1 交叉验证

**定义**：对比原文和幻灯片内容，检查要点数量是否一致。

**验证方法**：
```
原文要点数 = 幻灯片要点总和
```

**验证标准**：
- 要点数量完全一致
- 要点内容完全保留
- 要点顺序基本保持

---

### 5.2 数据点验证

**定义**：检查文档中的所有数据点是否完整保留在幻灯片中。

**验证方法**：
```
原文数据点数 = 幻灯片保留数据点数
```

**验证标准**：
- 所有数字完全一致
- 所有百分比完全一致
- 所有统计数据完全一致

**注意**：数据点是决策依据，必须完整保留，不能四舍五入或近似处理。

---

### 5.3 特殊内容验证

**定义**：检查表格、图表、引用等特殊内容块是否正确处理。

**验证内容**：
- 表格：数量、位置、内容完整性
- 图表：类型、位置、数据准确性
- 引用：来源标注、内容一致性

---

## 6 模板相关术语

### 6.1 模板文件

**定义**：预定义的HTML文件结构，包含页面布局、样式和组件。

**命名规则**：`XX-template-name.html`

**常用模板**：
- `01-cover-page.html` - 封面页
- `08-table-of-contents.html` - 目录页
- `11-chapter-overview.html` - 章节首页
- `02-two-column-comparison.html` - 两列对比
- `03-three-column.html` - 三列布局
- `05-chart-text.html` - 图表加文字
- `99-closing.html` - 结束页

---

### 6.2 模板参数

**定义**：模板中需要根据实际内容填充的可变部分。

**常见参数**：
- `{{title}}` - 页面标题
- `{{subtitle}}` - 副标题
- `{{content}}` - 内容区域
- `{{date}}` - 日期
- `{{author}}` - 作者

---

## 7 样式相关术语

### 7.1 列表样式

**定义**：控制列表元素视觉呈现的CSS样式集合。

**类型**：
- 标准要点列表（bullet-list）
- 编号列表（numbered-list）
- 卡片列表（card-list）
- 对比列表（comparison-list）
- 时间线列表（timeline-list）
- 图标列表（icon-list）

---

### 7.2 CSS类名

**定义**：用于标识和引用HTML元素的类名称。

**命名规范**：
- 使用小写字母
- 使用连字符（-）分隔单词
- 语义化命名

**示例类名**：
- `.bullet-list` - 要点列表
- `.numbered-list` - 编号列表
- `.card-list` - 卡片列表
- `.comparison-list` - 对比列表

---

### 7.3 设计系统

**定义**：统一的设计规范和组件库，确保视觉一致性。

**包含内容**：
- 配色方案
- 字体规范
- 间距规范
- 组件样式
- 响应式规则

---

## 8 流程相关术语

### 8.1 步骤2.1

**定义**：文档结构识别阶段，目标是完整识别文档的所有标题层级和内容块。

**主要任务**：
- 提取标题层级（H1/H2/H3）
- 统计内容密度
- 识别特殊内容块

**输出产物**：文档结构地图、内容密度统计表、特殊内容标记清单。

---

### 8.2 步骤2.2

**定义**：页面类型规划阶段，目标是根据文档结构规划所有幻灯片页面的类型和数量。

**主要任务**：
- 规划固定页面（封面页、目录页、结束页）
- 规划章节首页
- 规划内容页数量

**输出产物**：完整页面规划清单、页面总数统计。

---

### 8.3 步骤2.3

**定义**：内容拆分阶段，目标是按照步骤2.2的规划，将内容拆分到具体页面。

**主要任务**：
- 分析内容元素
- 选择布局类型
- 分配内容到页面

**输出产物**：每一页的详细内容清单。

---

### 8.4 步骤2.4

**定义**：完整性验证阶段，目标是验证拆分后的内容是否完整保留了原文信息。

**主要任务**：
- 交叉验证（要点数量）
- 数据点验证（数据准确性）
- 特殊内容验证（表格、图表）

**输出产物**：内容完整性验证报告。

---

### 8.5 步骤2.5

**定义**：输出清单阶段，目标是生成完整的页面内容清单，为步骤3的HTML生成提供输入。

**主要任务**：
- 整理页面统计信息
- 详细记录每页内容
- 索引特殊内容

**输出产物**：幻灯片页面内容清单。
