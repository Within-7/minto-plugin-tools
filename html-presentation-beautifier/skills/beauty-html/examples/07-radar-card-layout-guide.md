# 雷达图+卡片布局示例 (07-radar-card-layout.html)

## 📋 示例概述

**文件名**: 07-radar-card-layout.html
**布局类型**: L6 复合布局 (雷达图 + 信息卡片)
**适用场景**: 能力评估、多维度分析、SWOT分析延伸
**匹配度**: 90%

---

## 🎯 适用场景

### 最佳使用场景
1. **能力评估**: 企业能力、个人能力、产品能力多维度评估
2. **维度分析**: 5-8个维度的雷达图可视化
3. **SWOT延伸**: 在SWOT基础上增加量化评分
4. **竞争分析**: 多个竞争对手的能力对比
5. **项目评估**: 项目健康度、风险评估等

### 内容特征
- **雷达图**: 5-8个评估维度,每个维度0-100分
- **总体评价**: 200-300字的综合评价文字
- **信息卡片**: 3个结构化信息卡片(优势/劣势/洞察)

---

## 🎨 设计特点

### 布局结构
```
┌─────────────────────────────────────────────────┐
│ 1.1 人维度:消费者导向的市场能力                 │ (页眉)
├──────────────────┬──────────────────────────────┤
│                  │  ┌─────────────────────────┐ │
│   雷达图         │  │ 核心优势 (绿色图标)    │ │
│   (400px高)      │  └─────────────────────────┘ │
│                  │  ┌─────────────────────────┐ │
├──────────────────┤  │ 待提升能力 (红色图标)  │ │
│ 总体评价         │  └─────────────────────────┘ │
│ (白色卡片+红边)  │  ┌─────────────────────────┐ │
│                  │  │ 关键洞察 (橙色图标)    │ │
│                  │  └─────────────────────────┘ │
└──────────────────┴──────────────────────────────┘
```

### 颜色系统
- **背景渐变**: `linear-gradient(135deg, #667eea 0%, #764ba2 100%)`
- **主强调色**: `#e74c3c` (雷达图、页眉下划线、总评左边框)
- **成功绿**: `#52c41a` (核心优势图标)
- **警告红**: `#ff4d4f` (待提升能力图标)
- **信息橙**: `#fa8c16` (关键洞察图标)

### 字体规范
- **页眉**: 24px, 600, #1a1a1a
- **卡片标题**: 16px, 600, #1a1a1a
- **卡片内容**: 13-14px, Regular, #666

## 📊 雷达图配置

### ECharts 雷达图参数

```javascript
var option = {
    radar: {
        indicator: [
            { name: '品牌管理能力', max: 100 },
            { name: '渠道建设能力', max: 100 },
            { name: '消费者洞察能力', max: 100 },
            { name: '用户关系管理', max: 100 },
            { name: '市场响应速度', max: 100 }
        ],
        shape: 'polygon',      // 多边形
        splitNumber: 4,        // 分割层数
        axisName: {
            color: '#666',
            fontSize: 12
        }
    },
    series: [{
        type: 'radar',
        data: [{
            value: [60, 70, 50, 45, 55],
            name: '能力评估',
            areaStyle: {
                color: 'rgba(231, 76, 60, 0.2)'  // 主强调色半透明
            },
            lineStyle: {
                color: '#e74c3c',
                width: 2
            }
        }]
    }]
};
```

### 维度数量建议
- **5个维度**: 最常见,视觉平衡
- **6个维度**: 适中,信息丰富
- **7-8个维度**: 信息密集,需要更大空间

## 🔧 使用方法

### 步骤1: 识别适用场景
```markdown
✅ 是否有5-8个评估维度?
✅ 是否需要量化评分(0-100)?
✅ 是否需要配套文字说明(优势/劣势/洞察)?
✅ 是否需要专业的能力评估展示?

如果以上全部为"是",使用此示例。
```

### 步骤2: 准备内容
```markdown
1. 雷达图数据:
   - 维度名称 × 5-8个
   - 每个维度的评分 (0-100)

2. 总体评价:
   - 200-300字综合评价

3. 信息卡片 × 3个:
   - 核心优势 (2-4个要点)
   - 待提升能力 (3-5个要点)
   - 关键洞察 (100-150字)
```

### 步骤3: 替换内容
```html
<!-- 1. 替换页眉标题 -->
<div class="header">1.1 人维度:消费者导向的市场能力</div>

<!-- 2. 替换雷达图数据 -->
indicator: [
    { name: '维度1', max: 100 },
    { name: '维度2', max: 100 },
    // ...
],
value: [60, 70, 50, 45, 55]  // 对应每个维度的分数

<!-- 3. 替换总体评价 -->
<div class="evaluation-content">
    [替换为实际评价内容]
</div>

<!-- 4. 替换信息卡片 -->
<div class="card-content">
    <ul>
        <li>[要点1]</li>
        <li>[要点2]</li>
    </ul>
</div>
```

## 💡 最佳实践

### 1. 雷达图维度设计
- **维度命名**: 简洁明确,5-8个汉字
- **评分标准**: 统一0-100分制
- **维度平衡**: 避免所有维度分数接近,需要有高低差异

### 2. 信息卡片内容
- **核心优势**: 2-4个要点,每个要点30-50字
- **待提升能力**: 3-5个要点,每个要点20-40字
- **关键洞察**: 1段100-150字的总结性文字

### 3. 视觉层级
- **雷达图**: 视觉重心,占据左侧400px高度
- **总体评价**: 次要重心,红色左边框强调
- **信息卡片**: 支撑信息,图标+标题+内容三层结构

### 4. 响应式适配
```css
/* 移动端适配建议 */
@media (max-width: 768px) {
    .main-content {
        flex-direction: column;  /* 改为纵向布局 */
    }
    .left-section, .right-section {
        flex: none;
        width: 100%;
    }
}
```

## 🚫 不适用场景

### 不要使用此示例的情况
1. **维度少于4个**: 使用柱状图或条形图
2. **维度多于8个**: 雷达图过于密集,考虑分组或其他图表
3. **无量化评分**: 使用纯文字卡片布局
4. **纯数据展示**: 使用表格或条形图
5. **时间序列数据**: 使用折线图或时间轴
