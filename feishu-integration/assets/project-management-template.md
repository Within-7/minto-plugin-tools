## 使用建议

1. **项目启动**
   - 创建项目记录，填写项目信息
   - 设置项目编号（建议格式：PRJ + 年份 + 序号）
   - 指定项目经理和优先级

2. **任务分解**
   - 将项目分解为多个任务
   - 每个任务关联到对应项目
   - 设置任务负责人和截止时间

3. **进度跟踪**
   - 定期更新任务状态
   - 记录实际工时
   - 根据任务完成情况更新项目进度

4. **团队协作**
   - 项目经理：full_access 权限
   - 任务负责人：edit 权限
   - 其他成员：view 权限

## 常见场景

### 场景 1: 创建新项目

```
1. 调用 create_bitable() 或 add_record() 创建项目
2. 设置项目状态为"未开始"
3. 分配项目经理
4. 设置项目起止日期
5. 分解项目为多个任务
```

### 场景 2: 分配任务

```
1. 在任务列表中创建新任务
2. 关联到对应项目
3. 设置任务负责人
4. 设置任务优先级和截止时间
5. 通知任务负责人
```

### 场景 3: 更新进度

```
1. 任务负责人更新任务状态
2. 填写实际工时
3. 项目经理根据任务完成情况更新项目进度
4. 项目进度 = (已完成任务数 / 总任务数) × 100%
```

- 范围：0-100

- 最小值：0

## 项目管理最佳实践

### 1. 项目编号规则

建议格式：`PRJ + 年份 + 序号`

示例：
- PRJ2024001
- PRJ2024002
- PRJ2024003

### 2. 任务分解原则

- 每个任务预计工时 ≤ 80 小时
- 任务应有明确的完成标准
- 任务之间应尽量独立
- 设置合理的任务依赖关系

### 3. 进度更新频率

- 每日更新：进行中的任务
- 每周更新：项目整体进度
- 每月回顾：已完成项目总结

### 4. 风险管理

- 项目进度 < 计划进度 50% → 标记为"已延期"
- 任务逾期未完成 → 提醒任务负责人
- 实际工时 > 预计工时 120% → 评估风险
