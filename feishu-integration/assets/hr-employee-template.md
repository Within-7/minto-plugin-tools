## 使用建议

1. **工号规则**
   - 建议格式：`EMP + 年份 + 序号`
   - 示例：EMP2023001, EMP2023002

2. **职级体系**
   - **技术岗（P系列）**：P1-P10
     - P1-P3：初级
     - P4-P6：中级
     - P7-P8：高级
     - P9-P10：专家
   - **管理岗（M系列）**：M1-M5
     - M1：主管
     - M2：经理
     - M3：总监
     - M4：VP
     - M5：CEO

3. **员工状态流转**
   - 试用期 → 在职（转正）
   - 在职 → 离职（辞职/辞退）

4. **考勤管理**
   - 每日记录考勤数据
   - 自动计算工作时长
   - 统计月度考勤情况

5. **协作权限**
   - HR 人员：full_access 权限
   - 部门经理：view 权限（查看本部门）
   - 员工本人：view 权限（仅查看自己）

## 常见场景

### 场景 1: 新员工入职

```
1. 调用 add_record() 创建员工档案
2. 生成工号（EMP + 年份 + 序号）
3. 填写基本信息
4. 设置员工状态为"试用期"
5. 设置试用期结束时间（通常 3-6 个月）
6. 通知相关部门（IT、行政等）
```

### 场景 2: 员工转正

```
1. 查询试用期员工列表
2. 评估试用期表现
3. 更新员工状态为"在职"
4. 可能调整职级和薪资
5. 记录转正时间
```

### 场景 3: 员工离职

```
1. 更新员工状态为"离职"
2. 记录离职日期
3. 记录离职原因
4. 办理交接手续
5. 停用相关账号权限
```

### 场景 4: 考勤统计

```
1. 按月统计考勤数据
2. 计算出勤率
3. 统计迟到、早退、请假次数
4. 生成月度考勤报表
5. 作为绩效考核参考
```

## 人事管理最佳实践

### 1. 员工档案管理

- 完整记录员工信息
- 及时更新员工状态
- 定期核对档案准确性
- 严格保密员工隐私

### 2. 考勤管理

- 每日记录考勤数据
- 定期生成考勤报表
- 异常考勤及时跟进
- 考勤数据与薪资挂钩

### 3. 薪资管理

- 准确计算薪资
- 保密薪资信息
- 按时发放工资
- 提供工资条

### 4. 员工发展

- 建立职级晋升通道
- 定期进行绩效评估
- 提供培训机会
- 规划职业发展路径
