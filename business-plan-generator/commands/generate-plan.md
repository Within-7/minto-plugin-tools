# generate-plan - 生成商业计划书

根据已创建的大纲和添加的数据,生成完整的商业计划书。

## 用法

```bash
/generate-plan
```

## 流程

1. 读取已保存的大纲
2. 检查是否有已添加的数据
3. **根据数据情况生成内容**：
   - **如果有数据**：根据大纲结构,填充数据内容,生成数据驱动的商业计划书
   - **如果没有数据**：基于大纲框架,生成通用商业计划书模板,使用占位符和示例数据
4. 生成markdown格式的商业计划书
5. 输出文件或显示内容
6. 提示用户可以后续添加数据重新生成

## 输出内容

### 商业计划书结构

根据行业和大纲,生成包含以下章节的完整商业计划书:

```markdown
# [行业名称]商业计划书

## 1. 执行摘要
[项目概述]
[核心亮点]
[融资需求]

## 2. 公司描述
[业务模式]
[愿景使命]
[法律结构]

## 3. 市场分析
[市场规模]
[目标客户]
[市场趋势]

## 4. 产品/服务
[产品介绍]
[独特卖点]
[竞争优势]

## 5. 市场推广策略
[定价策略]
[渠道策略]
[营销计划]

## 6. 运营计划
[团队结构]
[运营流程]
[里程碑]

## 7. 财务计划
[收入预测]
[成本分析]
[盈利模型]

## 8. 风险管理
[市场风险]
[运营风险]
[应对策略]
```

## 数据填充

根据用户通过add-data添加的数据,自动填充到相应章节:

### 市场数据填充
```
市场规模: 1000万
增长率: 15%
```
→ 市场分析部分:
   - 当前市场规模: 1000万
   - 未来3年预测: 1150万、1322万、1520万
   - 市场机会分析
```

### 竞品数据填充
```
竞品数量: 5个
```
→ 市场分析/产品部分:
   - 主要竞品分析
   - 竞品优势对比
   - 差异化策略
```

### 财务数据填充
```
初始投资: 200万
```
→ 财务计划部分:
   - 资金使用计划
   - 3年现金流预测
   - ROI分析
   - 盈亏平衡点
```

## 输出方式

### 选项1: 显示内容
直接在对话中显示生成的商业计划书

### 选项2: 保存文件
将商业计划书保存为markdown文件:
- 文件名: [行业]-business-plan-[日期].md
- 位置: 用户工作目录

### 选项3: 导出格式
提供多种导出格式:
- Markdown (默认)
- 可扩展支持PDF、Word等

## 注意事项

- 必须先运行/create-outline创建大纲
- 建议运行/add-data添加相关数据
- 生成的计划书基于大纲和数据
- 可以多次运行/generate-plan更新内容
