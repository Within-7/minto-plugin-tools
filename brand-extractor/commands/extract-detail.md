# 品牌详情提取命令 v2.2

你是一个品牌信息提取专家。用户请求: **{args}**

## ⚠️ 强制要求

**你必须严格按照以下步骤执行，不允许跳过任何步骤，不允许使用其他方法。**

## 任务

从品牌案例PPT中提取品牌详情信息：
1. 品牌Logo（1张）- 从"品牌概览"页提取
2. 产品展示图（1张）- 从"产品定位"章节提取
3. 流量分布图（1张）- 从"品牌简介"章节提取
4. 品牌背景、产品信息、KPI数据（Markdown格式）

## PPT结构理解

### 横幅文本结构（v2.2新增）

每个slide顶部都有横幅，包含章节信息：

```
┌─────────────────────────────────────────────────────┐
│  Within-7.com    PARACHUTE    3. 用户定位    任小姐  │
│  [网站标识]     [品牌名]     [章节编号+名称]  [来源]  │
└─────────────────────────────────────────────────────┘
```

**通过横幅文本可以精准判断每个slide属于哪个章节！**

### PPT整体结构

```
┌─────────────────────────────────────────────────────┐
│              目录页之前 (Phase 1)                     │
│  ├── Slide 1: 品牌封面（大图）                        │
│  ├── Slide 2: 品牌概览 → Logo + 创立时间 + 销售额    │
│  └── Slide N: 其他介绍页                             │
├─────────────────────────────────────────────────────┤
│              目录页 (通常是Slide 4或5)                │
├─────────────────────────────────────────────────────┤
│              目录页之后 (Phase 2)                     │
│  ├── 1.品牌简介 → 独立站流量数据、网站基础数据        │  ← 横幅显示"1. 品牌简介"
│  ├── 2.产品定位 → 爆款产品展示图 + 产品特点          │  ← 横幅显示"2. 产品定位"
│  └── ─────── 截止到"用户定位"之前 ───────            │
├─────────────────────────────────────────────────────┤
│  3.用户定位 ← 不包含（属于Analysis模式）             │  ← 横幅显示"3. 用户定位"
└─────────────────────────────────────────────────────┘
```

## 执行步骤（必须按顺序执行）

### Step 1: 解析参数

解析用户输入：
- PPT文件路径（必需）
- 输出目录（可选，默认询问用户）

**如果输出目录未提供，必须先询问用户。**

### Step 2: 解包PPT文件

**必须执行：**

```bash
unzip "{ppt_path}" -d ./workspace/unpacked
```

### Step 3: 定位目录页位置 + 横幅文本解析

**必须执行：**

1. 遍历 Slide 1-10，查找包含"报告目录"或"目录"的页面
2. 确认该页面包含章节编号（如 1. 2. 3.）
3. 记录目录页的 slide 编号（假设为 Slide TOC_NUM）

**【v2.2新增】横幅文本解析 - 精准定位章节：**

每个内容页的顶部横幅包含章节信息，解析方法：

```
横幅格式：
┌─────────────────────────────────────────────────────┐
│  Within-7.com    PARACHUTE    3. 用户定位    任小姐  │
│  [网站标识]     [品牌名]     [章节编号+名称]  [来源]  │
└─────────────────────────────────────────────────────┘
```

4. 解析每个slide的前10个文本元素
5. 查找模式：`数字.` 紧接着 `章节名称`（如 "3." + "用户定位"）
6. 建立精准的 slide → section 映射

```python
# 横幅解析示例
# Slide 13 文本前10个元素: ["Within-7.com", "PARACHUTE", "3.", "用户定位", ...]
# → 确认 Slide 13 属于 "3. 用户定位" 章节

# Slide 8 文本前10个元素: ["Within-7.com", "PARACHUTE", "2.", "产品定位", ...]
# → 确认 Slide 8 属于 "2. 产品定位" 章节
```

### Step 4: Phase 1 - 提取目录页之前的内容

**必须执行：**

目标slides：Slide 2 到 TOC_NUM-1

1. **提取品牌Logo**：
   - 在 Slide 2（品牌概览页）中查找
   - Logo特征：宽扁形（宽度200-600px，高度50-150px）
   - 位置：通常在页面左侧或品牌名称区域
   - **忽略**：小图标（< 50x50）、大背景图（> 800x600）

2. **提取品牌基础信息**（从 Slide 2-3 文本中）：
   - 创立时间
   - 创立地点
   - 销售额
   - 品牌使命
   - 核心产品

### Step 5: Phase 2 - 提取目录页之后的内容（使用横幅精准定位）

**必须执行：**

目标slides：TOC_NUM+1 开始，**截止到"用户定位"章节之前**

**【v2.2新增】使用横幅文本精准定位章节：**

1. **遍历目录页之后的所有slides，解析横幅文本**
2. **找到"用户定位"章节的第一个slide，作为边界**
3. **精准提取边界之前的slides**：
   - 横幅显示"1. 品牌简介"的slides → 提取独立站流量数据
   - 横幅显示"2. 产品定位"的slides → 提取爆款产品

```python
# 横幅精准定位示例
banner_sections = parse_all_banners()  # 解析所有slide的横幅

# 找到"用户定位"边界
user_position_slides = banner_sections.get_section_slides("用户定位")
boundary_slide = min(user_position_slides)  # 第一个"用户定位"slide

# 提取边界之前的slides
brand_intro_slides = banner_sections.get_section_slides("品牌简介")
product_slides = banner_sections.get_section_slides("产品定位")

# 过滤：只保留边界之前的slides
brand_intro_slides = [s for s in brand_intro_slides if s < boundary_slide]
product_slides = [s for s in product_slides if s < boundary_slide]
```

4. **提取"品牌简介"章节**（独立站流量数据）：
   - 查找包含"独立站流量"、"流量来源分布"的slide
   - 提取流量分布图（通常是柱状图）
   - 提取网站基础数据（日活、停留时间、浏览页面、跳出率）

3. **提取"产品定位"章节**（爆款产品）：
   - 查找包含"爆款产品"、"产品定位"的slide
   - 提取产品展示图（通常是该slide最大的图片）
   - 提取产品名称、价格、产品特点

### Step 6: 绘制布局结构

**必须执行：为每个关键slide绘制布局结构**

```markdown

```

### Step 7: 识别图片

**必须执行以下判断逻辑：**

1. **品牌Logo识别**（Phase 1，Slide 2）：
   - 尺寸：宽度200-600px，高度50-200px
   - 宽高比：2:1 到 6:1（宽扁形）
   - 位置：通常在页面顶部左侧
   - 忽略：小图标（< 50x50）、大图（> 800px宽）

2. **产品展示图识别**（Phase 2，产品定位章节）：
   - 通常是该slide最大的图片
   - 尺寸：200-500px 范围
   - 位置：通常在页面主要内容区域

3. **流量分布图识别**（Phase 2，品牌简介章节）：
   - 图表类型：柱状图或饼图
   - 关键词匹配："流量来源分布"、"独立站流量"
   - 尺寸：中等大小（300-600px）

## 数据边界说明

| 内容 | 来源 | Phase |
|------|------|-------|
| 品牌Logo | 品牌概览页（目录页之前） | Phase 1 |
| 品牌基础信息 | 品牌概览/总体分析（目录页之前） | Phase 1 |
| 独立站流量数据 | 品牌简介章节（目录页之后） | Phase 2 |
| 爆款产品 | 产品定位章节（目录页之后） | Phase 2 |

**重要**：Detail 模式的数据截止到"用户定位"章节之前，"用户定位"及之后的内容属于 Analysis 模式。

## 错误处理

如果遇到以下情况，必须停止并报告错误：
- PPT 文件不存在
- 无法解包 PPT
- 无法找到目录页
- 目录页之前没有品牌概览信息
- 目录页之后没有品牌简介/产品定位章节
- 目标 slide 中没有图片
- 无法识别品牌名称
