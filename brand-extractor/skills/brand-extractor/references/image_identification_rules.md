# 图片识别精确规则

本文档定义了如何精确识别PPT中各种类型图片的规则。

---

## 核心原则

1. **位置+尺寸+上下文** 三者结合判断
2. **排除法优先**：先排除明显不是目标的图片
3. **视觉确认**：必须实际查看图片内容确认

---

## 一、品牌Logo识别规则

### 1.1 排除规则（先排除这些）

| 排除条件 | 说明 |
|----------|------|
| `x < 50 AND y < 50 AND size < 50×50` | 左上角小图标（如Within-7.com的logo） |
| `size < 30×30` | 过小的装饰图标 |
| 文件名包含 `icon` | 通常不是品牌Logo |

### 1.2 识别规则

| 条件 | 判断 |
|------|------|
| `100 < x < 500` | 水平位置在品牌名区域 |
| `40 < y < 200` | 垂直位置在页面上方 |
| `150 < width < 500` | Logo通常是横向的，宽度适中 |
| `30 < height < 150` | Logo高度通常较小 |
| `width > height` | Logo通常是横向矩形 |
| **文本上下文** | 附近有"品牌概览"、"创立于"等文字 |

### 1.3 决策流程

```
1. 获取品牌概览页（Slide 2 或含"品牌概览"关键词的页面）所有图片
2. 过滤掉 x<50, y<50, size<50×50 的小图标
3. 在剩余图片中，找到位置在 (100-500, 40-200) 范围内的
4. 如果有多个候选：
   - 优先选择 width > height 的（横向Logo）
   - 优先选择尺寸较大的
5. 用Read工具查看图片内容确认
6. 与用户确认后提取
```

### 1.4 示例

```
图片候选：
- image2.png: pos=(7, 4), size=34×34  → 排除（x<50, y<50）
- image9.png: pos=(172, 60), size=327×79 → ✅ 符合所有条件
- image3.jpeg: pos=(680, 12), size=608×760 → 排除（背景图）

结果：image9.png 是品牌Logo
```

## 二、产品展示图识别规则

### 2.1 定位产品页

**关键词**：
- "爆款产品"
- "产品特点"
- "产品价格"
- "SKU"
- "设计"

### 2.2 识别规则

| 条件 | 判断 |
|------|------|
| `width > 200 AND height > 200` | 产品图通常较大 |
| 位于slide左侧或中部 | 产品图通常不在边缘 |
| 附近有产品文字 | "价格"、"SKU"等 |
| **不是图表** | 没有柱状/饼状特征 |

### 2.3 多产品图情况

如果产品页有多个产品图：
```
1. 左侧产品图：通常是主推产品
2. 右侧产品图：可能是对比产品
3. 都应该提取，命名为：
   - {brand}_product_1.png
   - {brand}_product_2.png
```

## 三、流量分布图识别规则

### 3.1 定位流量页

**关键词**：
- "独立站流量来源分布"
- "网站基础数据"
- "日活"
- "平均停留时间"
- "跳出率"

### 3.2 区分两种流量图

| 类型 | 特征 | 文件命名 |
|------|------|----------|
| **品牌详情-流量分布** | 柱状图，显示各渠道流量占比 | `{brand}_traffic_distribution.png` |
| **品牌分析-流量模型** | 流程图/漏斗图，显示流量转化流程 | `{brand}_traffic_model.png` |

### 3.3 柱状图识别

```
1. 找到包含"独立站流量来源分布"的slide
2. 找到该slide中最大的图片
3. 确认是柱状图（用Read查看）
4. 命名为 {brand}_traffic_distribution.png
```

## 四、用户定位饼图识别规则

### 4.1 定位用户定位页

**关键词**：
- "用户定位"
- "图例解释"
- "niche用户"
- "核心用户"

### 4.2 识别规则

| 条件 | 判断 |
|------|------|
| 包含多个圆圈/扇形 | 饼图/圆圈图特征 |
| 有"图例解释"文字 | 确认是用户定位图 |
| 尺寸中等 | 通常 500-900×400-700 |

### 4.3 避免混淆

```
用户定位饼图 vs 宠物定位饼图：
- 用户定位：标题是"用户定位"，圆圈代表用户类型
- 宠物定位：标题是"宠物定位"，圆圈代表宠物类型

务必确认标题文字！
```

## 五、外链图表识别规则

### 5.1 外链来源饼图

**定位关键词**："外链合作网站的类型"、"外链（冷流量）"

**特征**：
- 饼图/圆环图
- 显示不同网站类型占比
- 命名：`{brand}_backlink_source.png`

### 5.2 外链关键词柱状图

**定位关键词**："外链文章题目"、"热门关键词"

**特征**：
- **横向**柱状图
- 显示关键词和出现次数
- 命名：`{brand}_backlink_keywords.png`

### 5.3 避免混淆

```
外链柱状图 vs 网站截图：
- 柱状图：有明显的横向条形
- 网站截图：看起来像网页截图

用Read查看确认！不要仅凭尺寸判断。
```

## 六、社媒图片识别规则

### 6.1 KOL合作数据图

**定位关键词**："KOL合作数据"、"社媒渠道合作数据"

**特征**：
- 通常是表格或柱状图
- 显示各渠道KOL数量
- 命名：`{brand}_kol_data.png`

### 6.2 官方社媒截图

**定位关键词**："官方社媒"、"社媒数据"、"Instagram"、"Facebook"、"YouTube"

**识别**：
```
1. 找到包含社媒平台名的slide
2. 找到对应平台的截图（通常是Instagram/Facebook/YouTube界面）
3. 按平台命名：
   - {brand}_instagram.png
   - {brand}_facebook.png
   - {brand}_youtube.png
```

### 6.3 营销活动截图

**定位关键词**："活动概述"、"品牌活动"

**识别**：
```
1. 找到包含"活动"的slide
2. 提取活动相关的截图
3. 命名为 {brand}_activity_1.png, {brand}_activity_2.png ...
```

---

## 七、通用排除规则

以下类型的图片**通常不提取**：

| 类型 | 特征 |
|------|------|
| 装饰图标 | size < 50×50，位于行首 |
| 网站favicon | 极小，通常 < 20×20 |
| 分隔线图片 | 极窄（width < 5 或 height < 5） |
| 重复背景图 | 在多个slide中重复出现 |
| 页眉页脚logo | 固定位置，每个slide都有 |

---

## 八、识别检查清单

提取每张图片前，必须确认：

```
□ 是否在正确的slide页面？
□ 位置和尺寸是否符合目标图片特征？
□ 用Read工具查看了实际内容？
□ 与用户确认了图片对应关系？
□ 文件命名符合规范？
```
