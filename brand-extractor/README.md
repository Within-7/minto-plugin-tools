# Brand Extractor - PPT品牌信息提取器 v2.3

Claude Code 插件，从品牌案例PPT中自动提取结构化的品牌信息。

## v2.3 核心变化

### 文字关键词精准匹配图片

```
v2.2 方法：
定位"品牌简介"slides → 选最大的图片 → 可能选错

v2.3 方法：
定位"品牌简介"slides → 找包含"独立站流量来源分布"文字的slide → 该slide的图片就是流量图 ✓
```

### 匹配规则示例

| 图片类型 | 文字关键词 | 匹配逻辑 |
|---------|-----------|---------|
| **流量分布图** | "独立站流量来源分布"、"流量来源分布"、"日活"、"跳出率" | 找包含这些词的slide |
| **产品展示图** | "爆款"、"销量最好"、"产品系列"、"产品名称" | 找包含这些词的slide |
| **品牌Logo** | "品牌概览"、"创立于"、"品牌使命" | 找包含这些词的slide |

## v2.2 核心变化

### 横幅文本精准定位

```
v2.1 方法：
解析TOC目录结构 → 估算章节slide范围 → 可能不精确

v2.2 方法：
解析每个slide的横幅文本 → 精准判断该slide属于哪个章节 → 准确率高
```

### 横幅文本结构

```
┌─────────────────────────────────────────────────────┐
│  Within-7.com    PARACHUTE    3. 用户定位    任小姐  │
│  [网站标识]     [品牌名]     [章节编号+名称]  [来源]  │
└─────────────────────────────────────────────────────┘
```

通过解析横幅文本，可以精准定位每个slide属于哪个章节，无需依赖估算。

## v2.1 核心变化

### Detail 模式：两阶段提取方法

```
v2.0 方法：
解析TOC目录结构 → 只提取目录页之后的章节 → 遗漏目录页之前的Logo和基础信息

v2.1 方法：
Phase 1: 提取目录页之前（Logo + 品牌概览）
Phase 2: 提取目录页之后，截止到"用户定位"之前（独立站数据 + 爆款产品）
```

### PPT 结构理解

```
┌─────────────────────────────────────────────────────┐
│              目录页之前 (Slide 1~N)                   │
│  ├── Slide 1: 品牌封面大图                           │
│  ├── Slide 2: 品牌概览（Logo、创立时间、销售额）       │
│  ├── Slide 3: 品牌总体分析                           │
│  └── Slide N: 其他品牌介绍                           │
├─────────────────────────────────────────────────────┤
│              Slide N+1: 目录页（报告目录）            │
├─────────────────────────────────────────────────────┤
│              目录页之后                               │
│  ├── 1. 品牌简介 → 独立站流量数据                     │
│  ├── 2. 产品定位 → 爆款产品                          │
│  ├── 3. 用户定位 ← Detail模式截止这里                │
│  ├── 4. 流量模型                                     │
│  └── 5. 基石流量                                     │
└─────────────────────────────────────────────────────┘
```

### 核心改进

| 改进点 | v2.0 | v2.1 | v2.2 | v2.3 |
|--------|------|------|------|------|
| 章节定位 | TOC估算 | TOC估算 | 横幅文本精准定位 | 横幅 + 文字关键词 |
| 图片识别 | 靠文件名猜测 | 规则配置 | 规则配置 | **文字关键词优先匹配** |
| Logo提取 | 容易遗漏 | 品牌概览页 | 同v2.1 | 文字"品牌概览"定位 |
| 流量图识别 | 选最大的 | 选最大的 | 横幅定位 | 文字"独立站流量"定位 |
| 产品图识别 | 选最大的 | 选最大的 | 横幅定位 | 文字"爆款"定位 |
| 边界限定 | 无 | 截止用户定位前 | 横幅定界 | 同v2.2 |

## 功能特性

- 📋 **四种模式**：品牌目录、品牌详情、品牌分析、品牌社媒
- 🗂️ **目录驱动**：通过解析PPT目录结构精准定位内容
- 🔄 **两阶段提取**：目录页之前 + 目录页之后（Detail模式）
- 🖼️ **智能识别**：基于规则配置识别图片用途
- 📊 **数据结构化**：输出JSON或Markdown格式数据
- 📁 **规范输出**：按固定目录结构组织输出文件

```

```

## 四种模式对比

| 模式 | 数据范围 | 输出图片 | 数据格式 |
|------|---------|----------|----------|
| 品牌目录 | Slide 1（封面） | 1张 | JSON |
| 品牌详情 | 目录页之前 + 目录页后(截止到用户定位) | 3张 | Markdown |
| 品牌分析 | 用户定位、SEO、外链 | 6张 | JSON |
| 品牌社媒 | KOL、社媒、活动 | 7张 | JSON |

## 提取流程（v2.1）

### Catalog 模式
```
Step 1: 解包PPT
         ↓
Step 2: 定位 Slide 1（封面页）
         ↓
Step 3: 提取封面图 + 品牌名称
         ↓
Step 4: 回写JSON数据
```

### Detail 模式（两阶段）
```
Step 1: 解包PPT
         ↓
Step 2: 解析TOC（定位目录页位置）
         ↓
┌─────────────────────────────────────┐
│  Phase 1: 目录页之前                  │
│  ├── 提取品牌Logo（品牌概览页）        │
│  └── 提取品牌基础信息                 │
└─────────────────────────────────────┘
         ↓
┌─────────────────────────────────────┐
│  Phase 2: 目录页之后                  │
│  ├── 1.品牌简介 → 独立站流量数据       │
│  ├── 2.产品定位 → 爆款产品图          │
│  └── 截止到"用户定位"之前             │
└─────────────────────────────────────┘
         ↓
Step 3: 复制图片到目标目录
         ↓
Step 4: 回写Markdown数据
```

## 核心模块说明

### extractor.py - 主提取器
- 协调各模块完成提取流程
- 支持四种模式的参数解析
- 提供命令行入口

### toc_parser.py - TOC解析器
- 定位TOC页（通常是Slide 4）
- 解析目录结构，提取章节名称和层级
- 估算每个章节的slide范围

### image_identifier.py - 图片识别器
- 根据规则配置识别图片用途
- 支持多种图片类型：cover/logo/product/pie/bar/flow/screenshot
- 基于位置、尺寸、图表类型判断

### writer.py - 数据回写模块
- 支持JSON和Markdown格式输出
- 保持与v1.x兼容的输出格式
- 自动创建目录结构
